  Preview Mode

Your HTML file, including any associated Javascript files, has been dynamically loaded into the preview panel.

You can now use the integrated browser debugger to investigate and debug your code.

Hint: enter the construct
      debugger;
into your Javascript to force the browser debugger to halt execution in your code when the web debugger is active.

Source code editing is disabled during preview mode, as well as save functionality.

The recommended approach to the
    develop publish test debug
cycle is to open a separate web browser on an attached monitor, login, turn Retain Form in Memory off, and use the debugger in that session. This will allow your changes to be dynamically loaded at run-time giving the optimum development efficiency.

  Manage all your custom files here. You can drag and drop existing files into this folder tree. Click on any .js/.html file to open it in the code editor.

You can edit javascript (.js) files where you can write custom business logic to build any type of application functionality.
You can edit hypertext markup language (.html) files where you design forms to provide a user interface for any type of application functionality.
When you are editing a file, the Toolbox will appear beneath the Folders/Files tree. Select a tool to open the Code Generator where you can configure the generated JS/HTML before testing it, and/or inserting the code into your edited custom file. The Field Picker tool in the .html toolbox allows you to pick any field from your database, and program it for read/write operations. Entities and fields are managed in the Administrator Console. This fully integrated developer studio geatly improves your productivity and is unrivalled by other rapid application development (RAD) systems.
Here is a high-level summary of the developer studio process:

a.   Edit your source code (.js) and markup (.html) files using developer studio.
b.   Publish your changes to Apex which injects your code and markup into your running instance.
c.   Your code can call any existing functions in any exposed namespaces. Your markup inherits the base CSS of the Apex application.
d.   Your code can call the web API to pull and push data to your database and file storage.
e.   Your code can read and update your SQL Server database via the web API.
f.   Your code can read and update your G:\ drive files and folders via the web API.
